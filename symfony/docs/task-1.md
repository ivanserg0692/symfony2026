# Task 1

## RU

### Название
Запуск Swagger-документации и подготовка базового News API

### Описание задачи
На текущем этапе планируется запустить Swagger для API-маршрутов проекта Symfony, чтобы получить удобную и понятную точку входа для просмотра и проверки REST endpoints.

После настройки Swagger следующим шагом будет реализация базового API для работы с новостями. В минимальный объем войдут:
- структура versioned routes в формате `/api/v1`
- базовые endpoints для новостей
- подготовка DTO/response-формата
- описание endpoints в Swagger/OpenAPI

### Цель
Сначала зафиксировать и поднять документацию API, после чего перейти к разработке первой прикладной функциональности: API для работы с новостями.

## EN

### Title
Swagger setup and baseline News API preparation

### Task Description
At this stage, the plan is to launch Swagger for the Symfony API routes to provide a clear entry point for browsing, validating, and testing REST endpoints.

After Swagger is in place, the next step will be to implement a baseline API for working with news. The initial scope includes:
- versioned routes under `/api/v1`
- basic endpoints for news
- DTO/response format preparation
- Swagger/OpenAPI documentation for the endpoints

### Goal
First, establish and document the API layer with Swagger. After that, move to the first business feature: a basic API for news management.
